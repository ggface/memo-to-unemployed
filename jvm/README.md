# Java Virtual Machine

### GC Roots (Корневые точки)
- Локальные переменные в стеке (методы, вызовы функций).
- Статические поля классов.
- Активные потоки (Thread).
- Ссылки в JNI (нативные методы).
- Загруженные системные классы (например, Class-объекты).

Любой объект, недостижимый от корней, считается недостижимым. Это мусор, и его память будет освобождена.

Молодое поколение: Сборка мусора здесь происходит часто и очень быстро

Старое поколение: Сюда перемещаются объекты, которые пережили несколько сборок мусора в Young Generation. Сборка мусора здесь происходит реже.

### Stack
Хранит примитивы и ссылки на объекты в рамках выполнения методов.

Работает по LIFO (Last In, First Out).

Быстрый доступ, но фиксированный размер (может привести к StackOverflowError при глубокой рекурсии).

Каждый поток имеет свой стек.

### Heap
Хранит объекты (экземпляры классов, массивы).

Динамически расширяется (но ограничен размером памяти устройства).

Общий для всех потоков, требует синхронизации.

Подвержен фрагментации (но в ART это решено лучше, чем в Dalvik).

### Memory Leaks
Статические ссылки на Activity/Fragment

Неотписанные слушатели (RxJava, LiveData, BroadcastReceiver)

Анонимные классы, держащие ссылку на внешний контекст
